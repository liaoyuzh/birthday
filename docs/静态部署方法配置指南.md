# 静态部署方法 
 

## 概述

本项目支持通过**纯静态部署**方式使用 AI 图片优化功能。不需要 Cloudflare Workers 或 Node.js 服务端，AI API 调用直接在浏览器中完成。

## 工作原理

```
┌─────────────┐     ┌─────────────┐     ┌──────────────────┐
│   用户浏览器  │ ──► │  静态服务器   │ ──► │   火山引擎 API     │
│  (浏览器端)   │     │  (Nginx)    │     │  (直接调用)        │
└─────────────┘     └─────────────┘     └──────────────────┘
                          │
                    无 API 路由需求
```

## 配置步骤

### 第 1 步：获取火山引擎 API 密钥

1. 访问 [火山引擎控制台](https://console.volcengine.com/)
2. 注册/登录账号
3. 进入 **访问密钥** 创建 Access Key
4. 获取 `VOLC_ACCESS_KEY_ID` 和 `VOLC_SECRET_ACCESS_KEY`

### 第 2 步：配置环境变量

编辑项目根目录的 `.env.local` 文件，添加以下内容：

```bash
# 火山引擎 API 密钥（供静态部署使用）
NEXT_PUBLIC_VOLC_ACCESS_KEY_ID=你的AccessKeyId
NEXT_PUBLIC_VOLC_SECRET_ACCESS_KEY=你的SecretAccessKey

# 启用静态部署模式
NEXT_PUBLIC_IS_STATIC_DEPLOYMENT=true

# 官方域名（可选，用于域名跳转）
NEXT_PUBLIC_OFFICIAL_DOMAIN=https://你的域名/
```

### 第 3 步：构建静态文件

```bash
npm install
npm run build
```

构建完成后，静态文件输出到 `out/` 目录。

### 第 4 步：部署到 Nginx

将 `out/` 目录下的所有文件部署到你的 Nginx 服务器：

```bash
# 示例：复制到服务器
scp -r out/* user@your-server:/var/www/perler-beads/
```

### 第 5 步：配置 Nginx

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /var/www/perler-beads;
    index index.html;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    gzip_min_length 1000;

    # 静态资源缓存
    location /_next/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # SPA fallback
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 图片资源缓存
    location /docs/ {
        expires 1y;
        add_header Cache-Control "public";
    }
}
```

### 第 6 步：配置 HTTPS（生产环境必须）

```bash
# 使用 Let's Encrypt 免费证书
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 环境变量说明

| 变量名 | 说明 | 必须 |
|--------|------|------|
| `NEXT_PUBLIC_VOLC_ACCESS_KEY_ID` | 火山引擎 Access Key ID | ✅ |
| `NEXT_PUBLIC_VOLC_SECRET_ACCESS_KEY` | 火山引擎 Secret Access Key | ✅ |
| `NEXT_PUBLIC_IS_STATIC_DEPLOYMENT` | 设为 `true` 启用静态部署模式 | ✅ |
| `NEXT_PUBLIC_OFFICIAL_DOMAIN` | 官方域名（用于非官方域名跳转） | 可选 |

## 安全说明

⚠️ **重要**：API 密钥会暴露在浏览器前端代码中。

**建议措施**：
1. 在火山引擎控制台创建**受限的 API Key**，只允许调用特定的视觉服务接口
2. 不要使用具有过高权限的密钥
3. 定期轮换密钥

## 架构说明

### 客户端 AI 调用流程

1. 用户选择图片并点击"AI 优化"
2. 图片在浏览器中压缩并转为 Base64
3. 使用 **Web Crypto API** (HMAC-SHA256) 生成签名
4. 直接向火山引擎 API 发送请求
5. 轮询任务状态直到完成
6. 返回优化后的图片 URL

### 关键文件

- [volcEngineClient.ts](file:///d:/perler-beads-master/src/lib/volcEngineClient.ts) - 火山引擎 API 客户端（浏览器端）
- [aiOptimize.ts](file:///d:/perler-beads-master/src/utils/aiOptimize.ts) - AI 优化入口，根据环境变量决定调用方式

### 条件判断逻辑

```typescript
// aiOptimize.ts 中的判断逻辑
if (IS_STATIC_DEPLOYMENT && VOLC_ACCESS_KEY_ID && VOLC_SECRET_ACCESS_KEY) {
  // 静态部署：直接调用火山引擎 API
  const result = await callVolcEngineAI(base64Image, prompt, accessKeyId, secretAccessKey, onProgress);
} else {
  // Cloudflare Workers/Vercel：调用本地 /api/ai-optimize
  const response = await fetch('/api/ai-optimize', { ... });
}
```

## 常见问题

### Q1: 构建后 AI 功能不工作

检查浏览器控制台是否有错误信息，常见原因：

1. 环境变量未正确配置
2. `NEXT_PUBLIC_IS_STATIC_DEPLOYMENT` 未设为 `true`
3. API 密钥无效或已过期

### Q2: 提示 "API request failed"

1. 检查网络连接
2. 确认火山引擎 API 密钥有效
3. 检查浏览器控制台的具体错误信息

### Q3: 图片未能通过安全检测

火山引擎返回 `IMAGE_RISK` 错误，表示图片内容不符合平台规范，请尝试其他图片。

## 一键部署命令

```bash
# 1. 安装依赖
npm install

# 2. 配置环境变量（编辑 .env.local）
# 3. 构建
npm run build

# 4. 部署到服务器
scp -r out/* user@your-server:/var/www/perler-beads/

# 5. SSH 到服务器重载 Nginx
ssh user@your-server
sudo systemctl reload nginx
```
